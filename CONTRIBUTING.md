# Contributing to Lido CSM Node Operator

Thank you for considering contributing! We welcome all types of contributions, including bug reports, documentation improvements, and feature requests.

## How to Contribute

1.  **Fork the repository** to your own GitHub account.
2.  **Clone the fork** to your local machine.
3.  **Create a new branch** for your changes.
4.  **Make your changes** and commit them with clear, descriptive messages.
5.  **Push your changes** to your fork.
6.  **Submit a Pull Request** (PR) to the `main` branch of this repository.

## Reporting Issues

If you find a bug or have a feature request, please [open an issue](https://github.com/your-username/lido-csm-operator/issues). Provide as much detail as possible, including:

- Steps to reproduce the issue.
- Expected behavior vs. actual behavior.
- Relevant log output or screenshots.

## Code Style

- Follow standard Ansible and YAML best practices.
- Ensure all new tasks are idempotent.
- Document any new variables or features in the relevant `.md` files.
